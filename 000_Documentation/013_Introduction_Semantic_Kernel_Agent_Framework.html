El Marco de Agente de Semantic Kernel

El marco de agente de Semantic Kernel fue creado para superar las deficiencias de los sistemas de agentes crudos, ofreciendo capacidades avanzadas como la conversación entre agentes y la ejecución paralela.

Características Clave

El marco se destaca por su orquestación modular, agentes orientados a objetivos con conciencia contextual, y una arquitectura flexible y extensible. La capacidad más notable es el "chat de grupo", que permite a los agentes colaborar para resolver tareas complejas.

Tipos de Agentes

Puedes crear tres tipos de agentes principales con este marco:

  • Agente de finalización de gráficos: Ideal para modelos de código abierto o entornos sin Azure AI Service.
  • Agente asistente OpenAI: Una opción si trabajas fuera de Azure.
  • Agente Azure AI: Se integra directamente con el Servicio de Agente de Azure AI para la gestión de agentes.

Componentes del Marco

El sistema se basa en un flujo de trabajo orquestado por el Núcleo (Kernel). Este orquesta la función de selección (para elegir al agente correcto), el chat de grupo (donde ocurre la interacción) y la función de terminación (para finalizar el proceso). Todo esto se apoya en la gestión de hilos para mantener el contexto de las conversaciones.

Relación con el Servicio de Agente de Azure AI

Es crucial entender que el marco de agente de Semantic Kernel no reemplaza a Azure Agent Service; lo complementa. El servicio de Azure es el lugar para definir y mantener tus agentes, mientras que el marco de Semantic Kernel te permite usarlos y orquestarlos en tu código personalizado para crear soluciones más avanzadas.
